# เอกสารสเปก — ระบบเว็บไซต์จัดการแข่งขันเกม

> ระบบจับคู่และจัดการการแข่งขันเกม (ตัวอย่างใช้เกมฟุตบอล) แบ่งเป็นลีค/รายการแข่งขัน
> แทนการจัดตารางด้วยมือและส่งผลผ่านแชท

---

## 1. ภาพรวม

เว็บไซต์สำหรับจัดการแข่งขันเกมที่แบ่งเป็น "ลีค" หรือ "รายการแข่งขัน" แต่ละลีคมีผู้เล่นตามจำนวนที่แอดมินกำหนด ผู้เล่นแข่งกันเองนอกเว็บ (นัดกันผ่านช่องทางติดต่อ) แล้วกลับมาบันทึกผลในเว็บ แอดมินเป็นผู้ดูแลและอนุมัติผลเมื่อจำเป็น

หลักการสำคัญที่สุดของระบบ (เป็นกระดูกสันหลังของทุกฟีเจอร์):
> **ผลแมตช์คือข้อเท็จจริงที่ไม่ถูกลบ — ตารางคะแนน อันดับ และสายการแข่งทั้งหมด "คำนวณสด" จากผลแมตช์ ส่วนการแพ้บาย/แก้ผล เป็น override ที่ถอดออกได้** ทำให้การแก้ไขและกู้คืนปลอดภัยเสมอ

---

## 2. บทบาทผู้ใช้

### แอดมิน
- มีคนเดียวทั้งระบบ ผู้ใช้คนแรกที่สมัครจะถูกตั้งเป็นแอดมินและเป็นแอดมินถาวร
- เมื่อเข้าสู่ระบบจะอยู่ที่หน้าแผงแอดมินเสมอ
- **แอดมินไม่ลงเล่นในลีคที่ตัวเองคุม** (เป็นจริงโดยอัตโนมัติเพราะมีแอดมินคนเดียว)
- หน้าที่: สร้าง/เริ่ม/ลบลีค, อนุมัติผลที่มีข้อพิพาท, กดแพ้บาย, ย้อนรอบถ้วย, เตะผู้เล่นออกก่อนเริ่ม

### ผู้เล่น
- สมัครบัญชี แล้วเข้าร่วมลีคที่เปิดรับสมัครได้เอง (ลีคเป็นแบบสาธารณะ)
- บันทึกผลการแข่งขันของตัวเอง, ติดต่อคู่แข่งเพื่อนัดแข่ง
- เล่นได้หลายลีคพร้อมกัน

## 3. การสมัคร บัญชี และความเป็นส่วนตัว

### ข้อมูลที่เก็บตอนสมัคร
| ข้อมูล | บังคับ | หมายเหตุ |
|---|---|---|
| ชื่อที่ใช้แสดง | ✅ | ชื่อที่โชว์ในลีค (แยกจากชื่อเฟซ) |
| อีเมล + รหัสผ่าน | ✅ | ใช้ล็อกอิน/กู้บัญชี (รองรับ Google sign-in ด้วยก็ได้) |
| เบอร์โทร | ✅ | ไว้ให้แอดมินติดต่อ |
| เฟซบุ๊ก (ลิงก์โปรไฟล์/username) | ✅ | เก็บเป็นลิงก์ ไม่ใช่แค่ชื่อ เพื่อกันติดต่อผิดคน |
| Line ID | ❌ | ออปชัน |
| รูปโปรไฟล์ | ❌ | ถ้าไม่ใส่ ระบบสุ่มไอคอนเริ่มต้นให้ (มีหลายแบบ) |

### การแก้ไขข้อมูล
ผู้ใช้แก้ข้อมูลส่วนตัวเองได้ทั้งหมด **ยกเว้นอีเมล** (อีเมลล็อกไว้)

### การมองเห็นข้อมูล (สำคัญมาก)
| ข้อมูล | ใครเห็น |
|---|---|
| ชื่อที่ใช้แสดง + รูปโปรไฟล์ | ทุกคน |
| เฟซบุ๊ก | คู่แข่งในแมทเดียวกัน + แอดมิน |
| เบอร์โทร + Line | แอดมินเท่านั้น |
| อีเมลล็อกอิน | ไม่โชว์ให้ใคร |

> เหตุผล: คู่แข่งเห็นชื่อ/ลิงก์เฟซก็พอจะยืนยันว่าติดต่อถูกคน (คอมมูนิตี้รู้จักกันบนเฟซอยู่แล้ว) ส่วนเบอร์โทรเก็บไว้ให้แอดมินใช้กรณีต้องประสานเรื่องแข่งใหม่

## 4. ลีค

### การตั้งค่าตอนสร้าง (แอดมินเลือก)
1. **ชื่อลีค**
2. **รูปแบบการแข่ง**: ชิงถ้วย (น็อคเอาท์) หรือ ลีคเก็บแต้ม
3. **รูปแบบนัด**: นัดเดียว หรือ เหย้า-เยือน *(เหย้า-เยือนใช้ได้เฉพาะลีคเก็บแต้ม — ถ้วยแข่งนัดเดียวเสมอ)*
4. **จำนวนผู้เล่น** (เงื่อนไขต่างกันตามรูปแบบ ดูข้อ 5)

### วงจรชีวิตลีค
```
เปิดรับสมัคร  →  (แอดมินกดเริ่ม → ล็อกรายชื่อ)  →  กำลังแข่ง  →  จบแล้ว
```

| สถานะ | ผู้เล่นเข้าร่วม | เตะผู้เล่น | ลบลีค |
|---|---|---|---|
| เปิดรับสมัคร | ได้ (กดเข้าร่วมเอง) / ออกเองได้ คืนสล็อต | ได้ | ได้ (ยกเลิก) |
| กำลังแข่ง | ไม่ได้ (ดูได้อย่างเดียว) | ไม่ได้ | **ไม่ได้** |
| จบแล้ว | — | — | ได้ (ลบถาวร) |

- ลีคเป็นแบบ **สาธารณะ** — มีหน้ารวมลีคที่เปิดรับสมัครให้ผู้เล่นกดเข้าร่วมเอง
- **คนที่ถูกเตะออก กลับเข้ามาใหม่ได้** (เผื่อมีสล็อตว่างและเขาอยากเล่นจริง)
- การลบเป็นการ **ลบถาวร** (ไม่มีระบบ archive)

## 5. รูปแบบการแข่ง

### 5.1 ชิงถ้วย (น็อคเอาท์)
- แพ้ตกรอบ ผู้ชนะเข้ารอบถัดไป
- **แข่งนัดเดียวเสมอ**
- **จำนวนผู้เล่นต้องเป็นเลขยกกำลังสองเท่านั้น** (4, 8, 16, 32, ...) ขนาดที่เลือก = ความจุของลีค
  - เต็มแล้วระบบล็อกรับสมัครอัตโนมัติ
  - ถ้าไม่เต็มจะกดเริ่มไม่ได้ (รอจนเต็ม หรือลบทิ้ง)
- **กรณีเสมอ → ยิงจุดโทษ**
  - ในหน้าส่งผลมีเช็คบ็อกซ์ "เข้าสู่การยิงจุดโทษ" จะกดได้ **เฉพาะเมื่อสกอร์เวลาปกติเสมอกัน**
  - เมื่อติ๊ก จะมีช่องกรอกสกอร์จุดโทษ และ **จุดโทษห้ามเท่ากัน** (ต้องมีผู้ชนะ)
- **การจับสาย**
  - รอบแรก: **สุ่มล้วน**
  - รอบถัดไป: **สายตายตัว** (ผู้ชนะคู่บน-ล่างเจอกัน รู้สายล่วงหน้าเหมือนบอลโลก)
- รอบถัดไปจะถูกสร้างก็ต่อเมื่อทุกคู่ในรอบปัจจุบันมีผลครบแล้ว

### 5.2 ลีคเก็บแต้ม
- **คะแนน**: ชนะ 3, เสมอ 1, แพ้ 0
- **จำนวนผู้เล่นต้องเป็นเลขคู่** — แอดมินกดเริ่มได้ที่เลขคู่ใดก็ได้ (ไม่ต้องเต็มตามเป้า) สามารถเตะคนออกให้เหลือเลขคู่ก่อนเริ่มได้
- **ตัวตัดสินเมื่อแต้มเท่ากัน** (เรียงตามลำดับ):
  1. แต้ม
  2. ผลต่างประตู (GD)
  3. ประตูได้ (GF)
  4. ถ้ายังเท่ากันเป๊ะ → แอดมินจัดแมตช์ตัดสินเอง 1 นัด (ใช้กติกาถ้วย: เสมอ → จุดโทษ เพื่อให้มีผู้ชนะแน่นอน)
- **เปิดทุกนัดพร้อมกัน** — ผู้เล่นเล่นกับใครก่อนก็ได้ ทยอยส่งผล

### 5.3 รูปแบบนัด
| รูปแบบ | จำนวนเกมต่อคน (ผู้เล่น N คน) | ตัวอย่าง 20 คน |
|---|---|---|
| นัดเดียว | N − 1 | 19 เกม |
| เหย้า-เยือน | 2 × (N − 1) | 38 เกม |

## 6. การส่งผลและการอนุมัติ

### ขั้นตอนปกติ
1. ผู้เล่นแข่งเสร็จ เข้าหน้าแมตช์ กรอกสกอร์ + แนบรูป (ไม่บังคับ)
2. การอนุมัติ:
   - **สองฝ่ายกรอกตรงกัน → อนุมัติอัตโนมัติทันที** (แอดมินดูย้อนหลังได้)
   - **กรอกไม่ตรงกัน → เข้าคิวแอดมิน** แอดมินดูทั้งสองสกอร์พร้อมป้าย "มีรูป/ไม่มีรูป" แล้วเลือก *ใช้ของฝั่งใดฝั่งหนึ่ง* หรือ *ให้แข่งใหม่*
3. เมื่ออนุมัติแล้ว: ถ้วย → ผู้ชนะเข้ารอบ / ลีค → ได้คะแนนตามเกณฑ์

### กฎเพิ่มเติม
- **ไม่มีเดดไลน์** (ตั้งใจ เพื่อเลี่ยงความยุ่งเรื่องเวลา)
- **แก้ไขผลที่ตัวเองส่งได้ "จนกว่าผลจะถูกอนุมัติ"**
  - แก้แล้วถ้าบังเอิญตรงกับคู่แข่ง → อนุมัติอัตโนมัติทันที
  - ถ้ายังไม่ตรง → ค้างรอแอดมินเหมือนเดิม
  - **อนุมัติแล้ว / ถ้วยขึ้นรอบถัดไปแล้ว → ล็อก** แก้เองไม่ได้ ต้องผ่านแอดมิน (กันแอบแก้ผลที่จบไปแล้ว)
- **รหัสแมตช์ (Match Code)**: ทุกแมตช์มีรหัสไม่ซ้ำ อ่านออกเสียงทางโทรศัพท์ได้ (เลี่ยงตัวสับสนอย่าง 0/O, 1/l) กดคัดลอกได้ แอดมินค้นหาด้วยรหัสได้
- มีปุ่ม **ติดต่อคู่แข่ง** ในหน้าแมตช์ (เปิดเข้าเฟซของคู่แข่ง)

## 7. กรณีพิเศษ

### 7.1 แพ้บาย (เฉพาะลีคเก็บแต้ม)
- แอดมินกดแพ้บายผู้เล่นรายบุคคล → **ทุกแมตช์ที่เกี่ยวกับคนนั้นกลายเป็นแพ้ 3-0** (เขียนทับทั้งแมตช์ที่แข่งแล้วและยังไม่แข่ง เพื่อความแฟร์ให้ทุกคู่ได้รับเท่ากัน)
- **แบนเฉพาะลีคนั้น** ลีคอื่นไม่กระทบ ผู้เล่นยังเล่นลีคอื่นได้
- **กู้คืนได้** (กรณีแบนผิดคน): เพราะตารางคำนวณสดจากผลแมตช์ + ป้ายแบนที่ถอดได้ → แค่ถอดป้ายแบน ตารางคำนวณใหม่ ทุกอย่างกลับมาเหมือนเดิมอัตโนมัติ (ผลจริงไม่เคยถูกลบ)

### 7.2 ติดต่อคู่แข่งไม่ได้ (เฉพาะถ้วย)
- ผู้เล่นกดปุ่ม "ติดต่อคู่แข่งไม่ได้" (**กดอีกครั้งเพื่อยกเลิกได้** หากติดต่อได้แล้ว)
- เมื่อกด → เข้าคิวแอดมิน แอดมินติดต่อทั้งสองฝั่ง
- ถ้าติดต่อฝ่ายใดฝ่ายหนึ่งไม่ได้จริง → อนุมัติให้ฝ่ายที่กดผ่านเข้ารอบ (บันทึกเป็น **"ชนะบาย"** ไม่ใช่สกอร์)
- **ทั้งคู่กดพร้อมกัน** → แอดมินเลือกผู้ผ่านเองหลังกดอนุมัติ (ถ้าติดต่อได้ทั้งคู่ ให้แข่งปกติ ปุ่มเป็นโมฆะ)

### 7.3 ย้อนรอบล่าสุด (เฉพาะถ้วย)
- ถ้วยไม่มีการแก้ผลแบบอิสระ แต่มีปุ่ม **"ย้อนรอบล่าสุด"** สำหรับแอดมินกันคลิกพลาด
- ใช้ได้เฉพาะ **รอบที่เพิ่งสร้างล่าสุด** เท่านั้น: ลบคู่ในรอบใหม่ทิ้ง กลับไปแก้ผลรอบก่อนหน้า แล้วสร้างรอบใหม่อีกที
- รอบที่ลึกกว่านั้นล็อกตาย

## 8. การแจ้งเตือน
- **กระดิ่งแจ้งเตือนในหน้าเว็บอย่างเดียว** (มีตัวเลขจำนวนที่ยังไม่อ่าน) — ไม่มี push, ไม่มีอีเมลแจ้งเตือน
- แจ้งเฉพาะเหตุการณ์ที่ต้องลงมือจริง ๆ เพื่อกันรก:
  - ผู้เล่น: คู่แข่งส่งผลแล้วรอคุณยืนยัน / ผลอนุมัติแล้ว / คุณเข้ารอบ / แอดมินขอให้แข่งใหม่
  - แอดมิน: มีข้อพิพาทหรือคำขอเข้าคิวใหม่
- อีเมลใช้แค่เรื่องบัญชี (ยืนยันอีเมล/รีเซ็ตรหัส) ซึ่ง Firebase Auth จัดการให้อัตโนมัติ

## 9. หน้าจอ (ออกแบบแบบมือถือเป็นหลัก)

### 9.1 หน้ารวมลีค
- การ์ดลีคที่เปิดรับสมัคร: ชื่อ, รูปแบบ (ถ้วย/เก็บแต้ม + นัดเดียว/เหย้าเยือน), จำนวนคน (เช่น 12/16), ปุ่มเข้าร่วม/เต็มแล้ว/เข้าร่วมแล้ว
- มีแท็บแยกดูลีคที่กำลังแข่ง / จบแล้ว (ดูอย่างเดียว)

### 9.2 หน้าหลักผู้เล่น
- **กล่อง "ต้องทำต่อ"** (ดึงจากทุกลีครวมกัน) — เฉพาะแมตช์ที่ถึงตาผู้เล่น เช่น คู่แข่งส่งผลรอยืนยัน, ถ้วยที่ถึงรอบแข่ง, แอดมินขอแข่งใหม่ พร้อมปุ่มติดต่อ + ส่งผล
- **กล่อง "ลีคของฉัน"** — แต่ละลีคบอกอันดับ/แต้มของตัวเอง + ความคืบหน้า + กางดูนัดทั้งหมดได้
- **สีสถานะของแต่ละนัด**:

| สี | ความหมาย |
|---|---|
| เทา | ยังไม่แข่ง |
| เหลือง | ส่งผลแล้วฝ่ายเดียว รออีกฝ่าย |
| ส้ม | สกอร์ไม่ตรง รอแอดมิน |
| เขียว | จบแล้ว (โชว์สกอร์) |

- แถบแจ้งเตือน

### 9.3 หน้าแมตช์ (ส่งผล)
- ข้อมูลแมตช์: รหัส (คัดลอกได้), ลีค/รอบ, คู่แข่ง, ปุ่มติดต่อคู่แข่ง
- ฟอร์มส่งผล: ช่องสกอร์ของทั้งสองทีม
- กล่องจุดโทษ (เฉพาะถ้วย, โผล่เมื่อสกอร์เสมอ, จุดโทษห้ามเท่ากัน)
- แนบรูป (ไม่บังคับ)
- ปุ่มส่งผล + ข้อความ "แก้ไขได้จนกว่าผลจะถูกอนุมัติ"
- สถานะการส่งผลของคู่แข่ง
- ปุ่ม "ติดต่อคู่แข่งไม่ได้" (เฉพาะถ้วย, กดยกเลิกได้)
- *ลีคเก็บแต้ม: ไม่มีกล่องจุดโทษ และไม่มีปุ่มติดต่อไม่ได้*

### 9.4 ตารางคะแนนลีค
- คอลัมน์: อันดับ, ทีม, แข่ง, **ได้**, **เสีย**, +/- (GD), แต้ม
- ไฮไลต์แถวของผู้เล่นเอง
- อธิบายเหตุผลการจัดอันดับเมื่อแต้ม/GD เท่ากัน

### 9.5 หน้าสายถ้วย
- สายตายตัว (ผู้ชนะคู่บน-ล่างเจอกัน)
- สกอร์จุดโทษแสดงในวงเล็บ เช่น 2 (4) = เสมอ 2-2 ชนะจุดโทษ 4-3
- แสดง "ชนะบาย" ให้รู้ว่าไม่ใช่ผลแข่งจริง
- ไฮไลต์รอบที่กำลังแข่ง + เส้นทางของผู้เล่น
- บนมือถือเลื่อนซ้าย-ขวาดูรอบถัดไปได้

### 9.6 แผงแอดมิน (แดชบอร์ดแบบแตะเข้าไปจัดการ)
- **แดชบอร์ดสรุป**: ตัวเลขสรุป (รอตัดสิน/แมตช์ค้าง/จำนวนลีค) มีตัวเลขแดงเตือนจำนวนที่ต้องลงมือ
- รายการแบบสั้น แตะเข้าไปจัดการทีละอัน (ไม่กางทุกปุ่มในหน้าเดียว)
- **ต้องจัดการ**: สกอร์ไม่ตรง (ตัดสิน), ติดต่อไม่ได้ของถ้วย (อนุมัติ/ยกเลิก), แมตช์ค้างไม่มีใครส่งผล (ข้อมูลประกอบการตัดสินใจ)
  - ตัวเลขแดงนับเฉพาะที่ต้องลงมือจริง (ข้อพิพาท + คำขอถ้วย) ไม่นับแมตช์ค้าง
- **จัดการลีค** (ปุ่มเปลี่ยนตามสถานะลีค): สร้างลีค, เริ่ม, เตะผู้เล่น, แพ้บาย + เลิกแพ้บาย, ย้อนรอบล่าสุด, ลบ, ดูตาราง/สายของลีคนั้น

## 10. เทคโนโลยีและหลักการข้อมูล

### เทคโนโลยี
- **ฐานข้อมูล/หลังบ้าน**: Firebase
  - **Firestore** — เก็บข้อมูลผู้ใช้ ลีค แมตช์ แจ้งเตือน
  - **Firebase Auth** — ล็อกอิน (อีเมล/รหัส และ/หรือ Google), ส่งเมลยืนยัน/รีเซ็ตอัตโนมัติ
  - **Firebase Storage** — เก็บรูปสกอร์ (ต้องเปิด Blaze plan ซึ่งมีโควต้าฟรีก้อนใหญ่)
- เว็บแอป responsive เน้นมือถือ

### หลักการข้อมูล (สำคัญ — ต้องฝังตั้งแต่วันแรก)
- **ผลแมตช์ = ข้อเท็จจริงที่ไม่ถูกลบ** เก็บผลที่ผู้เล่นส่งไว้เสมอ
- **ตารางคะแนน อันดับ และสายถ้วย = คำนวณสดจากผลแมตช์** ไม่เก็บคะแนนเป็นตัวเลขที่คอยบวก-ลบ
- **แพ้บาย / การแก้ผลของแอดมิน = override ที่ถอดออกได้** (เช่น ป้ายแบนต่อผู้เล่นต่อลีค) ถอดแล้วระบบคำนวณใหม่กลับเป็นเดิม
- ที่ขนาดคอมมูนิตี้นี้ การคำนวณตอนอ่าน (compute-on-read) เพียงพอ ไม่ต้องทำ pipeline ซับซ้อน

### โครงข้อมูล Firestore (แนวทางเริ่มต้น)
- `users` — โปรไฟล์, ข้อมูลติดต่อ, บทบาท (player/admin)
- `leagues` — ชื่อ, ประเภท (cup/points), รูปแบบนัด, ขนาด, สถานะ, รายชื่อผู้เล่น, ป้ายแบน (override)
- `matches` — leagueId, รอบ (ถ้วย), คู่แข่งสองฝั่ง, รหัสแมตช์, สถานะ, การส่งผลของแต่ละฝั่ง (สกอร์/จุดโทษ/รูป/เวลา), ผลที่อนุมัติ, ธงชนะบาย
- `notifications` — userId, ประเภท, อ้างอิงแมตช์/ลีค, อ่านแล้วหรือยัง

## 11. สรุปกฎ/Validation สำคัญ
- ถ้วย: จำนวนผู้เล่น ∈ {4, 8, 16, 32, ...} เท่านั้น; ลีคเก็บแต้ม: เลขคู่
- จุดโทษ: ติ๊กได้เฉพาะเมื่อสกอร์ปกติเสมอ; สกอร์จุดโทษห้ามเท่ากัน
- สกอร์: จำนวนเต็มบวกเท่านั้น
- ล็อกรายชื่อผู้เล่นเมื่อแอดมินกดเริ่มลีค
- แก้ผลเองได้เฉพาะก่อนอนุมัติ
- ลบลีคได้เฉพาะสถานะ "เปิดรับสมัคร" หรือ "จบแล้ว"
- การมองเห็นข้อมูลส่วนตัวตามตารางข้อ 3
